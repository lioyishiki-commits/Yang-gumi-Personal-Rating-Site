# **1. 一分钟认识 Yang-gumi**

Yang-gumi 是一个本地运行的 ACGN 个性化评分网站。它把你的动画、漫画、轻小说、游戏评分与 Bangumi 公开数据放在同一个界面中，方便建立私人档案、查看排行榜、对比大众评分，并维护季度新番与今日美图。

它的核心特点是“本地单用户”：主站默认只在本机 127.0.0.1:8501 打开，编辑、评分、备份都发生在你的电脑上。开源仓库只提供程序和默认资源，不附带任何作者个人数据。

| **你关心的问题**      | **直接答案**                                                                              |
| --------------- | ------------------------------------------------------------------------------------- |
| 我第一次打开会看到作者数据吗？ | 不会。首次启动会创建空数据库，需要你自己新增或从备份恢复。                                                         |
| 需要 Bangumi 账号吗？ | 不需要。程序只读取公开数据，无 OAuth，不写回 Bangumi 账户。                                                 |
| 会上传我的评分吗？       | 不会。主库是本地 SQLite；公开 JSON 也需要你主动导出。                                                     |
| 图片从哪里来？         | 今日美图默认读取你用户主目录下的 Pictures/Yang-gumi/Portrait 和 Pictures/Yang-gumi/Wallpaper，也可用环境变量改。 |
| 能给朋友看吗？         | 可以启用 8502 只读分享，但只建议可信局域网或私人组网，不建议公网。                                                  |

# **2. 架构与数据流**

普通用户只需要会双击启动脚本即可。维护者可以把系统理解为五层：启动器、页面层、业务服务、数据库与缓存、外部公开数据源。

| **层级** | **负责什么**                         | **典型文件或目录**                                                                 |
| ------ | -------------------------------- | --------------------------------------------------------------------------- |
| 启动器    | 创建虚拟环境、安装依赖、检查端口、打开浏览器。          | 安装并启动 Yang-gumi.bat、启动 Yang-gumi.bat、install_and_start.sh、start_yanggumi.py |
| 页面层    | 首页、条目库、Bangumi、排行榜、评分设置、数据管理等界面。 | app.py、app_public.py、ui_components.py、ui_settings.py                        |
| 业务服务   | 评分计算、Bangumi 读取、季度新番合并、今日美图扫描裁切。 | scoring.py、bangumi_client.py、seasonal_service.py、daily_art.py、filtering.py  |
| 本地数据   | 作品、标签、缓存元信息、备份、导出。               | data/、backups/、exports/、covers/、static/运行缓存                                 |
| 外部公开源  | 公开条目、排行榜、季度排期、公开校验资料。            | Bangumi、YUC Wiki、KissSub，及 ANN/AniList/MAL/AniDB 等校验来源                      |

# **3. 新电脑安装、启动与访问**

本章从“刚下载 ZIP 的普通用户”视角写。你不需要先创建数据库，也不需要准备作者的任何文件。

## **3.1 Windows 首次安装**

1. 安装 Python 3.13.x 64 位完整版（支持 3.10-3.14），勾选“Add python.exe to PATH”，并用 `py -3.13 --version` 验证。

2. 从 GitHub 下载项目 ZIP，解压到一个普通文件夹；不要直接在压缩包里运行。

3. 双击“安装并启动 Yang-gumi.bat”。脚本会检查 64 位 Python、创建 .venv、安装固定版本 requirements.txt、安装旧 Edge 兼容前端并启动网站。

4. 浏览器打开 [http://127.0.0.1:8501](http://127.0.0.1:8501) 后，即可开始使用空数据库。

Windows 首次启动：双击 安装并启动 Yang-gumi.bat

Windows 以后启动：双击 启动 Yang-gumi.bat

| **路径建议：**把仓库放在英文或简短中文路径下均可；不要放在需要管理员权限的系统目录中。公开说明中不需要写任何个人用户名或绝对路径。 |
| ------------------------------------------------------------------- |

## **3.2 Windows 日常启动**

首次安装成功后，以后只需要双击“启动 Yang-gumi.bat”。它会复用已创建的 .venv，不会每次重新安装全部依赖。主站默认 headless=true，由启动器统一打开浏览器，避免 Streamlit 和启动器各自打开一次造成“双开”。主站启动器不会启动 `share_public.py`；只有单独双击“启动只读分享.bat”才会开启 8502 只读服务。

## **3.3 macOS / Linux 安装启动**

macOS 与 Linux 使用 shell 脚本。首次使用前给脚本执行权限，然后运行脚本。

chmod +x install_and_start.sh

./install_and_start.sh

脚本会按项目配置安装依赖并启动服务。启动后在浏览器访问 [http://127.0.0.1:8501](http://127.0.0.1:8501)。

## **3.4 访问地址与端口**

## **3.5 首次启动后的空数据库**

开源仓库不会附带作者主库。第一次启动时，程序会自动在运行时数据目录中创建 SQLite 数据库和表结构，但作品、标签、评分、笔记都为空。你可以选择手动新增条目、用 Bangumi 搜索带入基础信息，或者在“数据管理”中恢复你自己的 SQLite 备份。

| **重要：**不要把 data/、backups/、exports/、static 缓存或个人封面提交到公开仓库。它们属于运行时内容。 |

## **4.1 首页**

首页是“私人档案仪表盘”。空数据库时，统计区会显示空状态；添加作品后才会出现最近添加、最近完成和评分概览。今日美图从你的本地图库读取；季度新番海报墙从公开来源合并后缓存显示。

* 今日美图：“竖屏”“壁纸”按钮用于重新选择对应文件夹，不是展示类型开关。

* 换一组：通常显示三张竖屏，10% 概率显示一张壁纸；抽中壁纸时显示“运气爆棚”火箭动画。

* 新番海报墙：每 10 秒自动移动，也可用左右箭头手动移动。

* 新番时光机：四栏同节拍刷新，帮助按季度查看播出安排。

## **4.2 条目库**

条目库只显示已经进入本地库的作品。它支持搜索、类型/子类型/状态/年份/评分/分差/标签筛选、排序、分页与进入详情。

| **分页位置** | **默认每页** | **可选值**                 |
| -------- | -------- | ----------------------- |
| 条目库      | 24       | 12 / 24 / 36 / 60       |
| Bangumi  | 24       | 8 / 12 / 16 / 24        |
| 排行榜、评分对比 | 24       | 12 / 24 / 50 / 100      |
| 标签作品     | 24       | 12 / 24 / 36 / 60 / 100 |

## **4.3 新增条目**

新增条目支持手工录入，也可以先用 Bangumi 查询带入公开基础信息，再补充自己的状态、评分和标签。首页季番过滤只影响首页展示；新增条目不禁止 TV、剧场版、OVA 等类型。

| **评分模式：**自动评分模式会根据子项公式计算总分；手动总分不会被评分设置的自动重算覆盖。 |
| ---------------------------------------------- |

## **4.4 Bangumi 页面**

Bangumi 页面读取 Bangumi 公开条目搜索和公开排行榜。它不要求账号绑定，没有 OAuth，不会把你的本地评分写回 Bangumi。

全新电脑尚无排行榜缓存时，页面会先显示“加载排行榜”按钮，不会在打开页面时阻塞等待网络；点击后才开始首次联网加载。

| **能力** | **说明**                   |
| ------ | ------------------------ |
| 搜索条目   | 使用公开 API，根据关键词回退搜索并读取详情。 |
| 公开排行榜  | 读取公开网页排行榜，并缓存到本地减少重复请求。  |
| 海报     | 优先使用本地缓存；远程图只作为后备。       |
| 权限     | 只读公开数据；不写回外部账户。          |

## **4.5 排行榜**

排行榜把个人数据按照总分、作品本体子项、Bangumi 分数、评分人数、公开名次、分差、完成时间等维度排序。Top10/20/50/100 等不同布局以页面可见选项和当前代码为准。

## **4.6 评分对比**

评分对比用于发现“Bangumi 高分但我无感”和“Bangumi 一般但我喜欢”的作品。进入这类列表的基本条件是我的分数与 Bangumi 公开分数的差值绝对值 ≥1.0，列表再按差距从大到小排列。

## **4.7 标签筛选与标签作品**

标签筛选页按标签聚合作品数、我的均分、Bangumi 均分与平均差。点击标签后进入标签作品列表，继续使用分页、排序和详情入口。

## **4.8 评分设置**

评分设置页可以调整三大项满分、子项权重和动态维度。保存或重置后，系统会重算所有 score_mode=auto 的历史作品；score_mode=manual 的手动总分不覆盖。

## **4.9 数据管理**

数据管理页负责备份、恢复、导出、批量同步、标签重建和质量检查。恢复前会自动备份当前主库，并要求二次确认。

| **功能**     | **用途**                     |
| ---------- | -------------------------- |
| 完整 JSON 导出 | 保留完整数据，用于个人备份或迁移。          |
| 公开 JSON 导出 | 删除私人备注、本地资源路径、封面本地路径等敏感信息。 |
| CSV 导出     | 打包为 ZIP，方便表格查看。            |
| SQLite 备份  | 最适合完整迁移；使用在线 backup API。   |
| 恢复         | 恢复 backups 目录中的备份；恢复前自动建立安全快照。 |
| 加载数据      | 上传卸载前保存或从其他电脑复制的 `.db`；验证完整性后替换当前数据。 |

# **5. 评分体系与计算公式**

评分输入范围为 0.0～10.0。各组只对已填写子项做加权归一化，未填写项不会当作 0 拉低分数。最终总分限制在 0～10，保留 2 位小数。

## **5.1 作品本体：最高 9.0 分**

| **子项** | **权重** | **对总分最高贡献** |
| ------ | ------ | ----------- |
| 剧情     | 40%    | 3.60        |
| 角色塑造   | 15%    | 1.35        |
| 作画/摄影  | 20%    | 1.80        |
| 演出     | 10%    | 0.90        |
| 音乐/配音  | 10%    | 0.90        |
| 节奏     | 5%     | 0.45        |

算法：已填子项的加权平均 ÷ 10 × 9.0。数据库列名 score_direction 是历史命名，当前实际含义是“演出”评分。

## **5.2 个人感受：最高 0.7 分**

| **子项**  | **权重** | **对总分最高贡献** |
| ------- | ------ | ----------- |
| 个人偏爱    | 40%    | 0.28        |
| 重看/重玩价值 | 20%    | 0.14        |
| 情绪后劲    | 20%    | 0.14        |
| 独特性     | 10%    | 0.07        |
| 氛围感     | 10%    | 0.07        |

没有填写任何个人感受子项时，该组得 0，不会自动补平均分。

## **5.3 时代加权：最高 0.3 分**

| **子项** | **权重** | **对总分最高贡献** | **显示条件**               |
| ------ | ------ | ----------- | ---------------------- |
| 影响力    | 60%    | 0.18        | Bangumi 评分人数严格大于 3,000 |
| 开创性    | 40%    | 0.12        | Bangumi 评分人数严格大于 3,000 |

| **关键规则：**时代加权输入只在 Bangumi 评分人数严格大于 3,000 时显示。没有填写时代加权子项时不产生加分。 |
| -------------------------------------------------------------- |

## **5.4 偏科惩罚**

偏科惩罚只统计作品本体已填写子项的最高分与最低分之差。它不受 Bangumi 评分人数影响，也不受“我的分数与 Bangumi 分差”影响。

| **作品本体最高-最低差值** | **扣分上限** | **实际扣分**       |
| --------------- | -------- | -------------- |
| ≤ 2.5           | 0        | 不扣分            |
| > 2.5           | 0.5      | 滑杆值 / 10 × 0.5 |
| > 3.0           | 1.0      | 滑杆值 / 10 × 1.0 |
| > 3.5           | 2.0      | 滑杆值 / 10 × 2.0 |

偏科惩罚滑杆范围为 0～10，最高只扣 2.0 分。

## **5.5 总分公式与示例**

| **步骤** | **公式**                               |
| ------ | ------------------------------------ |
| 扣分前    | 作品本体 + 个人感受 + 时代加权，并限制在 0～10。        |
| 最终分    | 扣分前 - 偏科惩罚，并限制在 0～10，保留 2 位小数。       |
| 自动重算   | 保存或重置评分设置后，重算全部 score_mode=auto 的作品。 |
| 手动总分   | score_mode=manual 不被自动重算覆盖。          |

示例：某作品本体加权后为 8.10，个人感受为 0.50，时代加权为 0.12，偏科惩罚为 0.30，则最终分 = 8.10 + 0.50 + 0.12 - 0.30 = 8.42。

| 绑定地址   | 0.0.0.0，用于同一局域网访问。                           |
| 令牌生成   | secrets.token_urlsafe(24)，通常约 32 字符。         |
| 令牌写法   | 文档只写生成方式和长度，不写真实值。                           |
| 数据库连接  | 只读连接读取主库。                                    |
| 禁用操作   | 修改、新增、删除、绑定、刷新公开数据等写操作。                      |
| 无权限提示  | 尝试写操作时提示“您没有操作权限”。                           |
| 安全定位   | 访问令牌只是轻量门槛，不是互联网级鉴权。                         |

| **警告：**不要把只读分享端口直接暴露到公网。推荐只在可信局域网、Tailscale/ZeroTier/WireGuard 等私人组网中使用。 |
| ------------------------------------------------------------------------ |

## **10.1 访客电脑怎么访问**

主人电脑保持只读分享程序开启后，访客电脑与主人电脑处于同一可信网络即可访问主人提供的只读链接。若发布包提供了访客启动器，访客端运行对应 BAT 即可，无需安装 Python；实际访问地址仍由主人端显示或提供。

## **10.2 公开导出与实时只读的区别**

| **方式**     | **数据来源**               | **适合场景**           |
| ---------- | ---------------------- | ------------------ |
| 公开 JSON 导出 | 手动导出的静态文件，删除私人备注和本地路径。 | 发给别人看固定快照。         |
| 只读分享站      | 直接以只读方式读取当前本地数据库。      | 在可信网络中查看接近实时的本地档案。 |

# **11. 备份、恢复、导入导出与换电脑迁移**

## **11.1 备份原则**

SQLite 备份使用 SQLite 在线 backup API，时间戳精确到微秒。备份和恢复前后都会检查必需表和数据库完整性。恢复是高风险操作，程序会先自动备份当前主库，并要求二次确认。

| **操作**       | **结果**                 |
| ------------ | ---------------------- |
| 创建 SQLite 备份 | 生成可完整恢复的数据库备份。         |
| 恢复 SQLite 备份 | 替换当前主库；恢复前自动备份当前主库。    |
| 完整 JSON      | 保存完整数据，适合个人留档。         |
| 公开 JSON      | 删除私人备注、本地资源路径、封面本地路径等。 |
| CSV ZIP      | 方便用表格软件查看或二次整理。        |

## **11.2 从旧电脑迁移到新电脑**

1. 在旧电脑打开 Yang-gumi 主站，进入“数据管理”。

2. 创建 SQLite 备份，并把备份文件复制到新电脑。

3. 在新电脑按第 3 章完成首次启动，让程序创建空数据库。

4. 进入新电脑“数据管理”，选择刚复制的 SQLite 备份并恢复。

5. 恢复后重新选择本地图库：竖屏目录、壁纸目录。

6. 如果旧电脑使用过自定义背景，在新电脑重新上传或重新选择背景。

7. 检查首页、条目库、评分设置、只读分享是否正常。

| **迁移重点：**数据库可以通过 SQLite 备份迁移；本地图库和自定义背景是电脑本地文件，不能假设新电脑路径相同，需要重新选择。 |
| ------------------------------------------------------------------ |

## **11.3 卸载前保存与重新加载**

1. Windows 双击“卸载 Yang-gumi.bat”。
2. 在弹窗中选择是否保存评分数据库；选择“取消”会停止卸载。
3. 选择保存时，必须把 `.db` 保存到 Yang-gumi 安装目录之外。程序会使用 SQLite 在线备份并执行完整性检查。
4. 保存完成后，卸载程序才会删除当前 Yang-gumi 安装目录；选择不保存时会再次提示数据不可恢复。
5. 以后重新安装并首次启动 Yang-gumi，进入“数据管理 → 加载已保存的数据”，上传该 `.db` 并勾选确认。
6. 加载前网站会自动备份当前数据库；验证失败的文件不会替换当前数据。

## **11.4 恢复后的检查**

| **检查项** | **通过标准**                       |
| ------- | ------------------------------ |
| 条目库     | 作品能正常分页、筛选、进入详情。               |
| 评分      | 自动评分作品显示总分；评分设置保存后 auto 作品可重算。 |
| Bangumi | 能搜索公开条目；网络失败时有清晰提示。            |
| 今日美图    | 重新选择目录后能换一组；壁纸有概率触发运气爆棚。       |
| 季度新番    | 海报墙不应因单一来源失败整体崩溃。              |
| 只读分享    | 可打开只读页；写操作被禁用。                 |

# **12. 错误保护、日志、故障排查与测试**

## **12.1 页面级错误保护**

各页面由 render_page_safely 保护。单页报错时，页面显示“页面暂时不可用”，导航仍可用，并把错误记录到 logs/app_errors.log，避免整个网站出现红色 Traceback。

## **12.2 常见故障排查**

| **现象**                | **优先处理**                                             |
| --------------------- | ---------------------------------------------------- |
| 双击 BAT 后窗口闪退          | 确认已安装 Python 3.13.x 64 位并加入 PATH；运行 `py -3.13 --version`，再在终端中运行脚本查看错误。            |
| 提示端口占用                | 关闭已有 Yang-gumi/Streamlit 进程，或检查 8501/8502 是否被其他程序占用。 |
| 依赖安装失败                | 检查网络、pip 源和 requirements.txt；必要时删除 .venv 后重新运行安装脚本。  |
| 浏览器打不开 127.0.0.1:8501 | 等待启动健康检查完成；若仍失败，查看启动窗口和 logs/app_errors.log。         |
| 新电脑出现白边或白框           | 先更新 Edge 并重启网站。程序已强制控件深色；旧 Edge 仍异常时安装 Dark Reader（扩展 ID `eimadpbcbfnmbkopoojfekhnkhdbieeh`），仅对 `127.0.0.1:8501` 启用并固定深色。 |
| 今日美图为空                | 确认所选目录及多级子目录中有 JPG/JPEG/JFIF/PNG/WebP/AVIF/BMP/GIF；点击“竖屏”“壁纸”重新选择目录，再点击“重新扫描图片”。页面会显示最近检查、识别、生成和无法读取的数量，便于判断目录或文件问题。 |
| 换一组不换图                | 可能候选太少或最近 30 个 key 防重复限制；增加图库后重建索引。                  |
| Bangumi 搜索失败          | 检查网络；公开 API 或网页临时异常时稍后重试。                            |
| 季度新番缺图                | 等待海报缓存补全；远程图失败时本地缓存会兜底。                              |
| 只读分享打不开               | 确认主人端分享站开启、端口 8502 未被拦截、双方在可信同一网络。                   |
| 恢复后数据不对               | 不要重复恢复未知备份；先使用恢复前自动备份回退。                             |
