### 口播

“看过几百部动画之后，最可怕的不是遇到总集篇，而是你突然想不起：这部我到底看没看过？当年为什么给它八分？那句短评又写到哪里去了？如果你的记录还散落在网站、表格和记事本里，今天介绍的 Yang-gumi，就是把这些世界线收束到自己电脑里的私人 ACGN 档案馆。”

## 00:35—01:20　项目定位与隐私边界

### 画面

- 首页全景，随后放大顶部九个导航入口。
- 屏幕字幕依次出现：`本地部署`、`单用户`、`数据默认不出电脑`、`Bangumi只读公开信息`。

### 口播

“Yang-gumi 是一个本地部署、单用户使用的 ACGN 评分与记录网站。动画、漫画、轻小说和游戏都能收录。作品状态、个人评分、标签、短评和私人笔记默认保存在本机 SQLite 数据库里。它读取 Bangumi 的公开条目信息时不需要登录，也不会修改你的 Bangumi 账号，更不会把个人评分上传回去。简单说，它更像你自己的迦勒底档案室，而不是另一个要求注册和同步的云平台。”

“它适合三类人：第一，想建立长期观看档案的人；第二，对评分维度有自己标准的人；第三，希望数据、图片和笔记掌握在自己手里的人。”

### 屏幕提示

`GitHub仓库只有程序，不包含作者的数据库、私人笔记、本地图片、令牌或本机路径。`

## 01:20—03:10　首页与今日美图

### 画面

1. 展示左侧档案卡：收藏、完成、在看、个人均分、Bangumi 均分和平均差。
2. 展示右侧“今日美图”，切换竖屏、壁纸和换一组。
3. 滚动到本季新番、季度记忆和评分画像。

### 口播

“首页先给你一张总览卡。收藏数量、完成数量、在看数量是一眼能懂的基础数据；个人均分和 Bangumi 均分放在一起，平均差则告诉你：你究竟是宽容的治愈系观众，还是每一帧都要审判的制作委员会。”

“右侧是今日美图。你可以分别选择新电脑自己的竖屏和壁纸文件夹，网站只建立本地索引，不会把图片上传。系统会识别多级子文件夹，并尽量围绕人物脸部和主体进行裁切。重新扫描使用增量索引，没变化的图片直接复用缓存，不必每次从第一张重新修炼。”

“继续向下是本季新番。它依据本机日期切换季度，主区域按规则只展示日本动画；缺失封面和精确 Bangumi 评分会在每日零点刷新。再下面的季度记忆，会把当前、五年前、十年前和二十年前同季作品放在一起——这时候你就会发现，芙莉莲觉得二十年很短，硬盘可不这么觉得。”

### 操作提示

- 录制“选择文件夹”时使用专门准备的演示图库。
- 文件夹选择结果属于当前电脑，不能把主电脑路径照搬到另一台电脑。
- 不要把“重新扫描”连续狂点；看到索引数量更新后再操作。

## 03:10—05:20　条目库、新增条目与详情页

### 画面

1. 打开“条目库”，演示搜索、类型、状态、年份、标签和评分区间筛选。
2. 打开“新增条目”，搜索一部大众熟悉的日本动画。
3. 展示搜索结果中的中文名、原名、封面、首播日期、两位小数评分与排名。
4. 进入详情页，展示状态、评分、短评、标签和笔记。

### 口播

“条目库是整个网站的档案柜。除了按名称搜索，还能按类型、状态、年份、标签、个人评分、Bangumi 评分以及双方分差筛选。作品多了以后，它比在几百行表格里施展搜索魔法轻松得多。”

“新增条目有两种方式。你可以手工填写，也可以搜索 Bangumi 公开条目。确认作品之后，封面、原名、首播日期、公开评分和排名会进入本地记录。Bangumi 评分统一精确到两位小数；这个精度不仅用于排行榜，也用于新增条目的搜索结果。”

“选中作品后，再填写自己的观看状态、评分、短评和标签。保存后，作品会出现在条目库、排行榜、评分对比和相关统计中。私人笔记只保存在本机，录视频时记得使用演示内容，不要把真正的黑历史做成全网同步。”

### 注意事项

- 搜索结果要核对中文名、原名、年份和封面，避免把重制版、续作或同名作品绑错。
- Bangumi 网络暂时不可用时，本地条目管理仍能使用；稍后再刷新公开信息即可。
- 演示删除条目会改变数据库，不建议在正式拍摄数据上操作。

## 05:20—06:40　个人评分体系

### 画面

- 打开“评分设置”，展示作品本体、个人感受、时代加权和偏科惩罚。
- 用一部演示作品展示分项变化如何影响总分。

### 口播

“Yang-gumi 的自动评分不是简单拍脑袋。默认结构是：作品本体最高九分，个人感受最高零点七分，时代影响与开创性最高零点三分，三部分合计十分。作品本体包含剧情、角色、作画或摄影、演出、音乐或配音和节奏；个人感受关注偏爱、后劲、独特性和氛围；时代项则更适合评价影响力与开创性。”

“如果不同核心维度差距过大，还可以启用偏科惩罚。比如作画已经到了剧场版，剧情却像被异世界卡车撞丢了两集，总分就不应该只靠一个长板起飞。所有权重都能在评分设置里调整；如果你已有明确结论，也可以使用手动总分。”

### 屏幕提示

`自动总分 = 作品本体 + 个人感受 + 时代加权 − 偏科惩罚，最终限制在0—10分。`

## 08:20—10:15　数据管理、备份与只读分享

### 画面

2. 展示 SQLite、JSON、CSV 的用途说明。
3. 简短展示“启动只读分享.bat”和8502端口示意图。

### 口播

“数据管理是保命回。SQLite 备份最完整，适合换电脑或长期保存；JSON 和 CSV 更适合查看或二次整理。重要更新前、批量修改前、换电脑前，都建议先做一次独立备份。”

“主站默认只在本机的 `127.0.0.1:8501` 运行。需要让同一局域网内的另一台设备查看时，可以单独启动只读分享，默认使用8502端口。访客只能看，不能改主人的档案。不要把8502直接映射到公网，也不要在视频里展示真实访问令牌、内网地址或防火墙规则细节。”

“这套边界很重要：程序可以从 GitHub 下载，但数据库、备份、封面缓存、图库、日志和令牌都不该上传 GitHub。否则私人档案馆就会一秒转职成公开处刑场。”

## 10:15—13:35　Windows 新电脑完整部署

### 画面与口播

#### 10:15—10:50　准备系统

画面：列出系统要求。

“新电脑建议使用64位 Windows 10 或 Windows 11，预留至少4GB空间。虚拟机建议至少分配两个处理器核心和4GB内存。普通 ZIP 用户不需要预装 Git、Node.js、Java或数据库。”

#### 10:50—11:35　安装 Python

画面：打开 Python 官方 Windows 下载页，展示安装界面的 PATH 选项。

“安装 Python 3.13.x 64位完整版。项目兼容 Python 3.10到3.14，但发布测试以3.13为基线。安装时一定勾选 `Add python.exe to PATH`。完成后打开命令提示符，输入 `py -3.13 --version`；能看到版本号再继续。”

屏幕链接：`https://www.python.org/downloads/windows/`

#### 11:35—12:20　下载仓库

画面：GitHub 的 Code → Download ZIP，完整解压到长期目录。

“在 GitHub 仓库点击 Code，再点 Download ZIP。下载后完整解压，不要直接在压缩包预览窗口里运行，也不要只复制其中一个 BAT。建议放到长期保留、路径清晰的文件夹。”

#### 12:20—13:05　首次安装并启动

画面：双击 `安装并启动 Yang-gumi.bat`，随后出现本地网页。

“第一次双击 `安装并启动 Yang-gumi.bat`。脚本会建立独立运行环境、安装依赖并启动网站。首次安装和第一次读取 Bangumi 数据需要联网，所需时间取决于网络。启动成功后，浏览器会打开 `http://127.0.0.1:8501`。”

#### 13:05—13:35　以后启动

画面：突出 `启动 Yang-gumi.bat`。

“以后使用只需双击 `启动 Yang-gumi.bat`。主站不会自动开启只读分享；只有你主动运行 `启动只读分享.bat`，8502服务才会出现。看到 Windows 防火墙提示时，主站只在本机使用通常无需开放公网权限。”

### 部署注意事项

- 所有文件必须完整解压；路径中不要使用压缩包内部的临时目录。
- 首次安装不要关闭命令窗口；如果长时间无进度，先看是否在等待网络下载。
- 代理或 VPN 异常只会影响外部公开数据，不应影响本地数据库的基本使用。
- Internet Explorer 不受支持；推荐 Chrome、Edge 或 Firefox。
- 不要从非官方镜像下载来历不明的 Python 安装包。

## 15:15—16:35　换电脑迁移与卸载

### 画面

- 用流程图展示：旧电脑创建 SQLite 备份 → 复制到新电脑 → 新电脑部署空站 → 数据管理恢复 → 重新选择本地图库。

### 口播

“换电脑时，不要复制浏览器缓存来碰运气。先在旧电脑创建 SQLite 备份，把备份文件单独复制到安全位置；在新电脑按刚才的步骤部署网站，再到数据管理恢复。恢复后检查收藏数量、评分、标签和详情页。”

“本地图片文件夹路径不会跨电脑自动继承。新电脑要重新选择自己的竖屏和壁纸文件夹，这正是为了避免把主电脑路径硬塞进虚拟机。”

“卸载时运行 `卸载 Yang-gumi.bat`，根据提示决定是否先把数据保存到安装目录之外。确认备份可用之前，不要手动删除整个目录。”

## 16:35—17:30　常见问题排查

6. “8501打不开：确认启动窗口仍在运行，并检查端口是否被其他程序占用。”

- 使用演示数据库，不展示真实私人笔记、评分历史或未公开收藏。

- 不在公开视频中演示真实数据库的删除、覆盖恢复或公开端口映射。

- 录制后按一次完整流程复核：下载、解压、安装、启动、新增、图库、备份。
