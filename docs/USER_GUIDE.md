# 使用说明

## 页面导览

- **首页**：个人统计、“今日美图”和当前档案概览。
- **条目库**：搜索、状态筛选、类型筛选、排序、查看详情与编辑。
- **新增条目**：先搜索 Bangumi，也可以完全手动建立条目。
- **Bangumi**：核对或刷新公开条目信息。
- **排行榜**：按个人总分生成综合及分类榜单。
- **评分对比**：比较个人分数与 Bangumi 公共分数。
- **标签筛选**：通过手动标签与 Bangumi 标签组合筛选。
- **评分设置**：调整自动评分维度、权重与规则。
- **数据管理**：导出、备份、恢复、加载卸载前保存的 `.db` 数据与完整性检查。

## 添加第一个条目

1. 打开“新增条目”，输入中文、日文或原始标题。
2. 在搜索结果中确认作品类型、日期和封面，避免同名误绑定。
3. 填写观看状态、日期、评分、短评和标签。
4. 保存后到“条目库”检查详情；之后可随时编辑。

Bangumi 只是公开元数据来源。个人评分、私人笔记、标签和观看状态只写入本地 SQLite 数据库。

全新电脑第一次进入 Bangumi 排行榜时会显示“加载排行榜”按钮；点击后通过 Bangumi 官方 `/v0/subjects` 接口建立本地缓存，打开页面本身不会被慢网络卡住。动画、漫画、小说、游戏分别使用官方类型与分类参数，不再依赖容易变化的排行榜网页 HTML。

## 今日美图

首页“竖屏”“壁纸”按钮用于重新选择本机文件夹，不是切换当前展示类型。“换一组”通常显示三张竖屏图，并有 10% 概率显示一张壁纸；壁纸出现时显示“运气爆棚”火箭动画。

索引只递归扫描你明确选择的本机目录及其多级子文件夹，最多深入 12 层、检查 12,000 个文件，并且不会跟随目录链接。支持 JPG、JPEG、JFIF、PNG、WEBP、AVIF、BMP、GIF。竖屏或壁纸目录中的可读图片会按照对应展示框自动裁切，不再因为原图方向不标准而被整个跳过。首页会显示检查数、识别数、成功生成数和无法读取数。程序会生成用于首页的 WebP 缓存并计算视觉焦点，原图不会复制进数据库。

## 备份习惯

- 大量录入前、恢复数据前、更新程序前各做一次 SQLite 备份。
- 把重要备份复制到项目目录之外或个人云盘。
- JSON/CSV 适合查看与分析；完整迁移优先使用 SQLite 备份。

## 卸载与重新加载数据

Windows 双击 `卸载 Yang-gumi.bat` 后，程序会询问是否先保存评分数据库。选择保存时，必须把 `.db` 保存到 Yang-gumi 安装目录之外；程序验证备份完整后才会继续卸载。以后重新安装网站，可在“数据管理 → 加载已保存的数据”上传这个 `.db`，确认后恢复；加载前会自动备份当前数据库。

## 旧 Edge 白框处理

程序已经强制文本框、数字框、选择框、展开栏和文件上传区使用深色样式。若非常旧的 Edge 仍把少量原生控件画成白色，先更新 Edge 并重新启动网站。仍异常时，可安装 Chrome Web Store 中扩展 ID 为 `eimadpbcbfnmbkopoojfekhnkhdbieeh` 的 Dark Reader，只对 `127.0.0.1:8501` 启用并固定为深色；不要同时启用其他同类深色扩展。

## 只读分享

Windows 双击 `启动只读分享.bat` 即可在可信局域网中启动只读页面；其他系统可运行 `python share_public.py`。访问令牌会写入本地 `data/`，不会进入 Git。只读分享不等于面向互联网的身份认证系统，不要把带令牌的链接公开发布。
